  • Initialize a git repository using git init

  • Branch Definition: A branch is an independent line of development that allows you to work on new features, fix bugs, or experiment without affecting the main codebase

Staging, preparing our changes to be synchronized with our repository, is done using the git add and git commit commands.

To create a new branch use command git branch <branch-name>

Commands

git init
Initialize a new Git repository
git config --global user.name 'My Name'
Set the global username for Git commits
git branch <branch-name>
Create a new branch
git add <file>
Add changes to the staging area
git status
Show the working directory status
git commit -m "commit message"
Commit changes with a message
git config --global init.defaultBranch <branch-name>
Set the default branch name for new repositories
git checkout <branch-name>
Switch to a different branch
git log
View the commit history
git log --oneline
View the commit history in a condensed format
git branch -d <branch-name>
Soft delete a merged branch
git branch -D <branch-name>
Force delete a branch
git checkout <commit hash>
Switch to a specific commit

Rules for commit messages

  1. Commit messages must be concise
  2. Commit message should be direct and imperative

Your commit messages should sound more like an answer to the question "if applied to the codebase this commit will ____________"

Edge cases

Creating a new branch without first adding and committing changes can lead to confusion and potential loss of work.
